This article exercises every MDX component shipped with DevfolioX, and confirms code highlighting works across languages.

## Callouts

<Callout kind="info" title="Heads up">
Use callouts to draw attention without yelling. They render with an icon, color tint, and optional title.
</Callout>

<Callout kind="tip" title="Pro tip">
You can mix **markdown** and `code` inside callouts.
</Callout>

<Callout kind="warning">
Plain warning callouts work without a title.
</Callout>

<Callout kind="success" title="Shipped">
Done, deployed, validated.
</Callout>

<Callout kind="danger" title="Don't do this">
Avoid putting secrets in `.env` files that end up in git.
</Callout>

## Code blocks

TypeScript, with line numbers:

```tsx showLineNumbers
import { MDXRemote } from "next-mdx-remote/rsc";

export async function MdxRenderer({ source }: { source: string }) {
  return <MDXRemote source={source} />;
}
```

Bash:

```bash
npm run cf:build && npm run cf:deploy
```

Python:

```python
def hello(name: str) -> str:
    return f"hi, {name}"
```

A fenced block with no language (must not throw):

```
just plain text — no highlighting needed.
```

Inline `code` should also style nicely.
